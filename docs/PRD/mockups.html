Auth flow — TOTP rollout

Generated by Claude Code on 2026-04-19 · 02-spec.md @ e7ab31f

Goals

  • Replace session cookies on /v2/auth with JWT + TOTP second factor.
  • Migrate 14k active sessions without forcing re-login.
  • Preserve magic-link flow as fallback for users without TOTP set up.

Non-goals

  • Changing v1 auth. Kept in place behind a feature flag until next quarter.
  • Adding WebAuthn. Tracked separately.

Open questions

  1. Q1: Should auth flow support magic links as primary, or only as fallback after TOTP?
  2. Q2: Do we expire the session-cookie grace period at 7 or 14 days?
  3. Q3: Session store — assume Redis, or read from config?
  4. Q4: Do we rate-limit TOTP attempts per account or per IP?
